# Extend route inventory

This directory defines the canonical page inventory for the plugin author and extension route.

## Route order

1. Plugin Development overview
2. Create a Plugin
3. Hook Execution
4. Config Augmentation
5. Official Plugin References

## Page inventory

### Plugin Development

- `plugin-development/index.md` — route entry and boundary between plugin usage and plugin authoring
- `plugin-development/create-a-plugin.md` — minimal path to author a plugin
- `plugin-development/hook-execution.md` — lifecycle and hook sequencing model
- `plugin-development/config-augmentation.md` — how plugins extend config safely and predictably
- `plugin-development/official-plugin-references.md` — how to use official plugins as implementation references without turning consumer docs into author docs

## Naming notes

- `official-plugin-references.md` is intentionally author-facing and should not duplicate the consumer plugin guides in Learn.
- This route replaces the older habit of hiding plugin authoring under a mixed Advanced bucket.
